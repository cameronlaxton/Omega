# Omega — Current Handoff

## Project identity

Omega is a sports analytics platform with:
- a Monte Carlo simulation engine covering 9 sport archetypes (70+ leagues)
- a sandbox-runnable single-file deterministic engine (`omega_lite_standalone.py`) that an LLM agent imports at session start
- a self-healing agent runtime defined entirely in `prompts/system_prompt.txt`: WebSearch the missing slots, inject, re-run, refuse cleanly when sourcing fails
- a strategy and backtest system with frozen artifact support
- a profile-driven calibration learning loop
- an observer-style skills system (trace-recorder, evidence-validator, data-quality-grader)

The repo follows a bounded-autonomy model:
- the LLM controls reasoning, planning, evidence gathering (via its own WebSearch tool), arbitration, and explanation
- the deterministic engine owns simulation, calibration, backtesting, edge logic, staking, and `trace_id` minting

### Phase 6b: Frozen artifacts (COMPLETE — API-independence sub-part retired with the evidence layer)
- `omega/strategy/artifacts.py` — `FrozenArtifact` Pydantic model: typed, versioned, deterministic `artifact_id` via sha256 of event identity; decision-time data only; outcome attached separately at grading time.
- `trace_to_artifact()` and `compat_dict_to_artifact()` converters.
- `omega/strategy/backtest/engine.py` — `run()` accepts `List[FrozenArtifact]` or legacy `List[dict]`; collects `trace_ids` into `BacktestResult`.
- `omega/strategy/models.py` — `BacktestResult` gains `artifact_schema_version`, `calibration_policy`, `trace_ids`.
- `tests/strategy/test_artifacts.py` — 16 tests passing.

### Phase 6c: Calibration learning (COMPLETE)
- `omega/core/calibration/profiles.py` — `CalibrationProfile` Pydantic model + `ProfileStatus` enum (CANDIDATE → PRODUCTION → ARCHIVED/REJECTED).
- `omega/core/calibration/registry.py` — `CalibrationRegistry` JSON-backed storage, atomic writes, `promote()` auto-archives the incumbent.
- `omega/core/calibration/fitter.py` — isotonic (PAV) + shrinkage grid search; `evaluate()` returns Brier / ECE / log-loss; `compare()` recommends promotion.
- `omega/core/calibration/probability.py` — `apply_calibration()` accepts `league` and dispatches to the production profile via registry; static fallback when `league=None`.
- `omega/core/contracts/service.py` and `omega/strategy/backtest/engine.py` thread `league` through every calibration call site.
- `tests/core/test_calibration_profiles.py` — 22 tests passing.

## Constraints reminder

- Do not duplicate edge, calibration, staking, or grading logic in a second path.
- Every persistence format must be versioned.
- Outcome attachment must happen after initial trace persistence, not by mutating source records.
- The sandbox engine must remain network-free.
- Anti-overengineering: does this directly prevent bad sim inputs, bad recommendations, bad replay, or bad backtests? If no, defer it.
