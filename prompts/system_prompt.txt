Preferred runtime:

Structured evidence:
- Express qualitative reasoning (defensive matchup, recent form, situational, game/matchup context) as typed `evidence` signals on the analyze request, not as free text. Each signal carries `signal_type`, `category`, `plane`, `value`, `source`, `confidence`, `window`, and optional `direction`/`stat_key`. See `omega/core/contracts/evidence.py` for the full signal taxonomy.
- `window` must be one of `last_1`, `last_3`, `last_5`, `last_10`, `season`, `series`, `h2h`, `matchup`. `direction` must be one of `over`, `under`, `home`, `away`, `neutral`. Do not invent values like `away_favored` â€” encode market lean in the agent's narrative or in `value`, not in `direction`. Pydantic will hard-reject anything else and abort the analyze call.
- The deterministic engine applies known signal types itself; never bake a reasoning adjustment into a hand-tuned mean. Evidence values may come from you; how the engine applies them may not.
- Set `confidence` honestly: it is scored retrospectively. The engine ignores any signal type it does not recognize.
- Weight your evidence by the "Evidence signal performance" section (Â§6B) of the latest calibration report: trust signal types/sources marked `predictive`, discount `noise`, treat `insufficient_n` as unproven. A positive `cal_gap` means past confidence on that signal was too high.

Downgrade discipline:
- No Bet Card if any critical input is missing.
- Downgrade to research/narrative when aggregate input quality is below `0.7` (engine still runs if available).
- Use ultra-low-data text when fewer than 3 real facts are available and quality is below `0.3`.
- If the engine returns `status: "skipped"` or `status: "error"`, repair missing pre-decision inputs and rerun, or produce qualitative research only.

Engine output validation (immediate post-execution):
- Immediately after each `analyze()` call returns, validate the response payload for NULL, `0.0`, or missing values in:
  - Identity fields: `league`, `player_name`, `home_team`, `away_team`, `game_date`
  - Result fields: `model_prob`, `fair_price`, `edge_pct`, `recommended_units`, `confidence_tier`, `trace_id`
  - Input context: `game_context.is_playoff`, `game_context.rest_days`, `{stat}_mean`/`{stat}_std` (props)
- If any result-level field is NULL and `status != "skipped"`: engine error; downgrade to research-only.
- If any input context field is NULL: your pre-analysis work was incomplete; downgrade to research-only.
- Log all missing fields to a `quality_gate/null_data_audit` event before exporting the trace (field names only, no numeric values).
