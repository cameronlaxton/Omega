# OMEGA_RUN_RECIPE

How to run Omega and feed its JSON output back into a Claude.ai Project. There is exactly one runtime path now: the sandbox engine plus the agent's WebSearch self-heal loop. The previous FastAPI service (Path B), all external-API collectors (The Odds API, Anthropic, Perplexity, ESPN, Redis), and the `[CLAUDE-ESTIMATED]` fallback have been retired.

If the sandbox engine is not available, the agent is in **QUALITATIVE MODE** for the entire session: matchup narrative, news, recent form, and listed sportsbook lines from cited sources — but NEVER edges, EV, Kelly, confidence tiers, BetSlips, or `trace_id`s. See [prompts/system_prompt.txt](prompts/system_prompt.txt) §3.

## `omega_lite_standalone.py` in the sandbox (no local install, no zip, no API keys)

Upload these two files to the Project's knowledge panel:
1. [`prompts/system_prompt.txt`](prompts/system_prompt.txt) — the master runtime instruction.
2. `omega_lite_standalone.py` — the single-file deterministic engine, generated by `python scripts/build_omega_lite.py --single-file`.

At session start the agent reads `omega_lite_standalone.py` from project knowledge, writes it byte-for-byte to its sandbox cwd, and imports it. There is no archive to extract; the previous `omega_lite-v1.zip` workflow is removed.

```python
# In the analysis tool, at session start (full install sequence in prompts/system_prompt.txt §3):
import os, sys
# (the agent has already written omega_lite_standalone.py to cwd from project knowledge)
sys.path.insert(0, os.getcwd())
from omega_lite_standalone import analyze

result = analyze({
    "home_team": "Boston Celtics",
    "away_team": "Indiana Pacers",
    "league": "NBA",
    "n_iterations": 5000,
    "seed": 42,
    "home_context": {"off_rating": 118.0, "def_rating": 108.0, "pace": 100.0},
    "away_context": {"off_rating": 115.0, "def_rating": 110.0, "pace": 98.0},
    "odds": {"moneyline_home": -160, "moneyline_away": 140, "over_under": 226.5},
})
# result["trace_id"] is "sandbox-XXXX"
# result["result"] is the full GameAnalysisResponse shape
```

For player props (the anchor-parlay use case) — see [OMEGA_LITE.md](OMEGA_LITE.md) for one runnable example per sport (NBA, MLB, NHL, NFL, EPL, ATP, PGA, UFC, CS2). The schema is identical across sports; only the `prop_type` and `player_context` keys vary. Example:

```python
result = analyze({
    "player_name": "Jayson Tatum",
    "league": "NBA",
    "prop_type": "pts",
    "line": 27.5,
    "odds_over": -115,
    "odds_under": -105,
    "player_context": {"pts_mean": 28.4, "pts_std": 6.2},
    "n_iterations": 5000,
    "seed": 42,
})
```

Read [OMEGA_LITE.md](OMEGA_LITE.md) for the per-sport coverage matrix.

## Data sourcing — the agent's WebSearch self-heal loop

The sandbox engine performs **no** network calls; it only does Monte Carlo + calibration + edge + Kelly math. Live data — sportsbook lines, player game logs, team ratings, news — is sourced by the **agent** (Claude.ai Project, ChatGPT Project, etc.) using its built-in WebSearch tool. There are no API keys to configure.

The full search → inject → re-run loop is specified in [prompts/system_prompt.txt](prompts/system_prompt.txt) §6. The short version:

1. Run `analyze(request)`.
2. If the engine returns `status: "skipped"` or `quality_gate.downgrades` contains `"dropped_bet_card"`, read `result["missing_requirements"]`.
3. For each missing slot, WebSearch the appropriate source: official sportsbooks (DraftKings, FanDuel, BetMGM, Caesars) for odds; `*-reference.com` and league sites for player and team stats.
4. Inject the values back into the request and retry. Bounded to 3 retries per analysis.
5. Exit conditions for refusal: market not available after 3 sportsbook searches; player stats not sourceable after 3 reference searches; retry budget exhausted with the quality gate still firm.

The agent must echo source URLs and timestamps for every injected value (system prompt §8).

## Copying Omega output into a Claude.ai Project

1. Open the Project chat.
2. Ask the question. The agent runs the install sequence, runs `analyze(...)`, runs the self-heal loop if needed, and renders.
3. The response cites the `sandbox-` `trace_id` from the engine return value and lists `Inputs used` with sources.

If you see a Project response that emits a Bet Card, edge%, EV%, Kelly, or a `trace_id` without an attached `omega_lite_standalone.py` smoke-test echo, that response is fabricated — refuse it and reinstall the engine.

## Local development of the engine

The math layer in `omega/core/` and the quality gate in `omega/reasoning/evaluator.py` are still source-controlled and tested. To rebuild the sandbox artifact after changing the math:

```bash
python scripts/build_omega_lite.py --single-file
pytest tests/omega_lite/
```

There is no FastAPI server to start, no API key to set. The `omega/` source tree exists purely so the sandbox artifact can be regenerated and parity-tested.
