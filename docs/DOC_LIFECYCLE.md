# Documentation Lifecycle

## Rule

Do not delete phase documentation just because the phase is complete.

## Keep and update

Keep:
- root `CLAUDE.md`
- local `CLAUDE.md`
- implemented phase specs
- reusable agent role briefs

Update when architecture changes.

## Archive in place

Phase docs should remain under their original phase folder with one status label:
- planned
- in progress
- implemented
- superseded

## Remove only if

A document should be removed only if it is:
- scratch-only
- fully duplicated elsewhere
- misleading and no longer salvageable
- replaced by a clearly superior canonical document

## Preferred pattern

- `docs/phase6/*`
- `docs/phase7/*`
- `docs/phase8/*`

Treat old phase docs as design history, not clutter.
