# Omega

A reasoning-led, quantitative sports analytics agent. Omega sits an LLM control layer (intent, planning, evidence gathering, explanation) on top of a deterministic engine (Monte Carlo simulation, calibration, contracts, staking) so betting recommendations are reproducible, auditable, and refuse cleanly when evidence is thin.

This is the canonical repository: `cameronlaxton/Omega`. The older repo `cameronlaxton/OmegaSports` is not the same project; do not use it as a reference.

## How Omega Runs Now

There is exactly one numeric runtime path: a deterministic engine plus an LLM/operator layer that gathers auditable inputs. In local Cowork/Codex operation, current odds resolve through The Odds API with BetMGM as the default bookmaker (`OMEGA_ODDS_API_KEY` in `.env` or the runtime environment). Multi-book fetches are reserved for explicit line-shopping, consensus, or audit requests. No-local Project agents still use cited WebSearch/WebFetch or ask the user to run the local resolver.

Two files drive no-local Project sessions:

1. [`prompts/system_prompt.txt`](prompts/system_prompt.txt): the master no-local runtime instruction. It forbids LLM-generated quantitative output and specifies the install sequence, input/output schemas, sport coverage matrix, evidence loop, and quality-gate response policy.
2. `omega_lite_standalone.py`: single-file deterministic engine generated by `python scripts/build_omega_lite.py --single-file`. Its math is parity-tested against the in-repo `omega_lite/` package.

Local agent environments should prefer `python -m omega.mcp.server`, including `omega_resolve_odds` for BetMGM-first odds resolution and the deterministic analyze tools for numeric output.

## What Lives In `omega/`

- `omega/core/`: Monte Carlo simulation, sport archetypes, probability calibration, request/response contracts, Kelly staking, and parlay math.
- `omega/reasoning/evaluator.py` and `gatherer.py`: quality-gate logic.
- `omega/integrations/odds_api.py`: The Odds API client for current BetMGM odds, explicit line-shopping requests, closing lines, and historical market snapshots.
- `omega/trace/`: trace persistence, bet records, closing lines, outcomes, CLV, and market snapshots for line movement.
- `omega/strategy/`: frozen artifacts, backtest engine, anchor-parlay scanner, and strategy versioning.
- `omega/mcp/`: typed local MCP tools over the deterministic contracts.

The previous `omega/api/`, `omega/evidence/`, `omega/reasoning/llm/`, and `omega/reasoning/orchestrator.py` have been removed.

## Quick Start

```bash
python -m venv .venv
# Windows: .venv\Scripts\activate
# macOS/Linux: source .venv/bin/activate
pip install -e .

python scripts/build_omega_lite.py --single-file
python -m pytest tests/ -v
```

## Local Odds Resolution

Normal local odds resolution uses BetMGM:

```bash
python scripts/resolve_odds.py --kind game --league NBA --home-team "Los Angeles Lakers" --away-team "Boston Celtics"
```

Use multi-book mode only when explicitly shopping lines or auditing consensus:

```bash
python scripts/resolve_odds.py --kind game --league NBA --event-id evt-id --line-shopping
```

The resolver prepares market inputs and provenance only. It does not compute probabilities, edge, EV, Kelly, staking, confidence tiers, or trace IDs.

## Working With Omega From An LLM

The LLM is not the pipeline. It gathers evidence, resolves odds through the local BetMGM-first Odds API tool when available, runs `omega_lite_standalone.analyze(...)` or the local MCP analyze tools, and renders the result. Anything claiming to run Omega without a `sandbox-` `trace_id` from a real engine call is hallucinated.

Required operating documents for an LLM consumer:

- [prompts/system_prompt.txt](prompts/system_prompt.txt): no-local runtime instruction.
- [OMEGA_COWORK.md](OMEGA_COWORK.md): local Cowork/runtime instruction.
- [OMEGA_DATA_SOURCES.md](OMEGA_DATA_SOURCES.md): per-slot sourcing recipe, including the local BetMGM-first Odds API resolver.
- [OMEGA_RUN_RECIPE.md](OMEGA_RUN_RECIPE.md): install sequence and sandbox operating model.
- [OMEGA_LITE.md](OMEGA_LITE.md): per-sport gallery, parity guarantee, sandbox honesty contract.
- [OMEGA_STRATEGY.md](OMEGA_STRATEGY.md): anchor parlay playbook.

## Testing

```bash
python -m pytest tests/ -v
```

Tests are network-free and do not spend Odds API quota. The `tests/omega_lite/` suite parity-tests both `omega_lite/` and the generated `omega_lite_standalone.py` against the canonical `omega/` math layer.
