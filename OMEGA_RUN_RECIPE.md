# OMEGA_RUN_RECIPE

How to run Omega and feed its JSON output back into a Claude.ai Project. There is exactly one numeric runtime path: a deterministic engine plus auditable input sourcing. In local Cowork/Codex sessions, current odds come from the BetMGM-default Odds API resolver. In no-local Project sessions, the agent still uses cited WebSearch/WebFetch evidence or asks the user to run the local resolver.

If the deterministic engine is not available, the agent is in **QUALITATIVE MODE** for the entire session: matchup narrative, news, recent form, and listed sportsbook lines from cited sources, but never edges, EV, Kelly, confidence tiers, BetSlips, or `trace_id`s.

## `omega_lite_standalone.py` In No-Local Sandboxes

Upload these two files to the Project knowledge panel:

1. [`prompts/system_prompt.txt`](prompts/system_prompt.txt): the master no-local runtime instruction.
2. `omega_lite_standalone.py`: the single-file deterministic engine, generated by `python scripts/build_omega_lite.py --single-file`.

At session start the agent reads `omega_lite_standalone.py`, writes it byte-for-byte to its sandbox cwd, imports it, and runs the smoke test described in the system prompt.

```python
import os, sys
sys.path.insert(0, os.getcwd())
from omega_lite_standalone import analyze

result = analyze({
    "home_team": "Boston Celtics",
    "away_team": "Indiana Pacers",
    "league": "NBA",
    "n_iterations": 5000,
    "seed": 42,
    "home_context": {"off_rating": 118.0, "def_rating": 108.0, "pace": 100.0},
    "away_context": {"off_rating": 115.0, "def_rating": 110.0, "pace": 98.0},
    "odds": {"moneyline_home": -160, "moneyline_away": 140, "over_under": 226.5},
})
```

For player props, see [OMEGA_LITE.md](OMEGA_LITE.md). The schema is identical across sports; only the `prop_type` and `player_context` keys vary.

## Local Odds Resolution

The deterministic engine performs no network calls. Local odds resolution is a separate input-prep step:

```bash
python scripts/resolve_odds.py --kind game --league NBA --home-team "Boston Celtics" --away-team "Indiana Pacers"
```

Defaults:

- Bookmaker: BetMGM (`betmgm`).
- Game markets: `h2h`, `spreads`, `totals`.
- Player props: event-level markets mapped from Omega `prop_type` where available.
- Multi-book mode: only when `--line-shopping` or `--all-books` is explicitly supplied.

The resolver emits engine-ready odds fields plus full normalized quotes and provenance. It does not compute probabilities, edge, EV, Kelly, staking, confidence tiers, or trace IDs.

## Self-Heal Loop

1. Run `analyze(request)`.
2. If the engine returns `status: "skipped"` or `quality_gate.downgrades` contains `"dropped_bet_card"`, read `result["missing_requirements"]`.
3. For odds, use the local resolver when available; otherwise use cited sportsbook WebSearch/WebFetch.
4. For player/team stats, use official league/reference sources and compute derived `std` values in Python.
5. Inject the values back into the request and retry, bounded to 3 retries per analysis.
6. If the market is unavailable or the quality gate still holds firm, produce a Research Report with no Bet Card.

The agent must echo source URLs or local resolver provenance and timestamps for every injected value.

## Copying Omega Output Into A Claude.ai Project

1. Open the Project chat.
2. Ask the question. The agent installs the sandbox engine, gathers/receives inputs, runs `analyze(...)`, and renders.
3. The response cites the `sandbox-` `trace_id` from the engine return value and lists `Inputs used` with sources.

If a Project response emits a Bet Card, edge%, EV%, Kelly, or a `trace_id` without a real engine call, treat it as fabricated and reinstall the engine.

## Local Development

```bash
python scripts/build_omega_lite.py --single-file
pytest tests/omega_lite/
```

Set `OMEGA_ODDS_API_KEY` only for local odds resolution, closing-line capture, historical backfill, and line-movement snapshots. The deterministic engine itself does not use the key.
