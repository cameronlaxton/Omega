# Omega

A reasoning-led, quantitative sports analytics agent. Omega sits an LLM control layer (intent, planning, evidence gathering via WebSearch, explanation) on top of a deterministic engine (Monte Carlo simulation, calibration, contracts, staking) so betting recommendations are reproducible, auditable, and refuse cleanly when evidence is thin.

This is the canonical repository — `cameronlaxton/Omega`. The older repo `cameronlaxton/OmegaSports` is not the same project; do not use it as a reference.

## How Omega runs now

There is exactly one runtime path: a sandbox-resident deterministic engine plus an LLM agent that gathers evidence with its built-in WebSearch tool. The previous FastAPI service, external-API collectors (The Odds API, Anthropic, Perplexity, ESPN), and `[CLAUDE-ESTIMATED]` fallback have all been retired. There are **no API keys to configure**.

Two files drive every session:

1. [`prompts/system_prompt.txt`](prompts/system_prompt.txt) — the master runtime instruction. Forbids LLM-generated quantitative output; specifies the install sequence, the input/output schemas inline, the per-sport coverage matrix, the self-heal WebSearch loop, and the quality-gate response policy.
2. `omega_lite_standalone.py` — single-file deterministic engine. Generated by `python scripts/build_omega_lite.py --single-file`. Bit-identical math to the in-repo `omega_lite/` package, parity-tested.

Upload both to a Claude.ai Project knowledge panel. At session start the agent reads `omega_lite_standalone.py`, writes it byte-for-byte to its sandbox cwd, imports it, runs a smoke test, and only then begins producing numeric output.

## What lives in `omega/`

The remaining `omega/` source tree is the math + quality-gate layer that gets built into the sandbox artifact:

- `omega/core/` — Monte Carlo simulation, sport archetypes (9 archetypes, 70+ leagues), probability calibration, request/response contracts, Kelly staking, parlay math.
- `omega/reasoning/evaluator.py` + `gatherer.py` — quality-gate logic (drops Bet Cards on missing critical inputs, downgrades native_sim → mixed → research).
- `omega/synthesis/composer.py` — response shape definitions.
- `omega/strategy/` — frozen artifacts, backtest engine, anchor-parlay scanner, strategy versioning. Network-free, deterministic, used for the quant evaluation plane (see [CLAUDE.md](CLAUDE.md)).
- `omega/skills/` — small auditing skills (evidence validator, data quality grader, trace recorder).
- `omega/trace/` — trace persistence (Phase 6 work).

The previous `omega/api/`, `omega/evidence/`, `omega/reasoning/llm/`, and `omega/reasoning/orchestrator.py` have been removed.

## Quick start

```bash
# Install (math + tests only; no API/server deps)
python -m venv .venv && source .venv/bin/activate  # .venv\Scripts\activate on Windows
pip install -e .

# Build the sandbox artifact
python scripts/build_omega_lite.py --single-file
# Produces omega_lite_standalone.py at repo root.

# Tests
python -m pytest tests/ -v
```

## Working with Omega from an LLM (Claude.ai Projects, etc.)

The LLM is *not* the pipeline — it gathers evidence with WebSearch, runs `omega_lite_standalone.analyze(...)`, and renders the result. Anything claiming to "run Omega" without a `sandbox-` `trace_id` from a real engine call is hallucinated.

Required operating documents for an LLM consumer:

- [prompts/system_prompt.txt](prompts/system_prompt.txt) — master runtime instruction.
- [OMEGA_HANDBOOK.md](OMEGA_HANDBOOK.md) — what Omega is, the architectural layers, 11 output packages, execution modes, 3 input importance tiers, quality-gate refusal logic.
- [OMEGA_DATA_SOURCES.md](OMEGA_DATA_SOURCES.md) — per-slot WebSearch recipe and source-credibility hierarchy. There are no API integrations; this document is the agent's sourcing playbook.
- [OMEGA_RUN_RECIPE.md](OMEGA_RUN_RECIPE.md) — install sequence, self-heal loop summary, sandbox-only operating model.
- [OMEGA_LITE.md](OMEGA_LITE.md) — per-sport gallery, parity guarantee, sandbox honesty contract.
- [OMEGA_STRATEGY.md](OMEGA_STRATEGY.md) — anchor parlay playbook.

## Testing

```bash
python -m pytest tests/ -v
```

Tests are network-free and require no API keys. The `tests/omega_lite/` suite parity-tests both `omega_lite/` and the generated `omega_lite_standalone.py` against the canonical `omega/` math layer.
