# Omega — Current Handoff

## Project identity

Omega is a sports analytics platform with:
- a Monte Carlo simulation engine covering 9 sport archetypes (70+ leagues)
- a sandbox-runnable single-file deterministic engine (`omega_lite_standalone.py`) that an LLM agent imports at session start
- a self-healing agent runtime defined entirely in `prompts/system_prompt.txt`: WebSearch the missing slots, inject, re-run, refuse cleanly when sourcing fails
- a strategy and backtest system with frozen artifact support
- a profile-driven calibration learning loop
- an observer-style skills system (trace-recorder, evidence-validator, data-quality-grader)

The repo follows a bounded-autonomy model:
- the LLM controls reasoning, planning, evidence gathering (via its own WebSearch tool), arbitration, and explanation
- the deterministic engine owns simulation, calibration, backtesting, edge logic, staking, and `trace_id` minting

## What's been completed

- Deleted `omega/api/`, `omega/evidence/`, `omega/reasoning/llm/`, `omega/reasoning/orchestrator.py`.
- Result: no FastAPI, no HTTP clients, no Redis. Evidence gathering is the agent's WebSearch loop.

### Skills system
- Observer-style `SkillBase` ABC with `observe()` wrapper that never raises.
- Three shipped skills: `trace-recorder` (enabled by default), `evidence-validator`, `data-quality-grader`.
- Skills config in `omega/skills/config.json`.

### Phase 6: Trace persistence, frozen artifacts, calibration learning (COMPLETE)
- **6a Trace persistence** — SQLite schema V5: traces, outcomes, bet_records, closing_lines, market_snapshots, schema_versions. `TraceStore` with full CRUD + CLV math.
- **6b Frozen artifacts** — `FrozenArtifact` Pydantic model; `trace_to_artifact()` + `compat_dict_to_artifact()` converters; backtest engine accepts `List[FrozenArtifact]`.
- **6c Calibration learning** — `CalibrationProfile` model, `CalibrationRegistry` (JSON-backed), `CalibrationFitter` (isotonic + shrinkage); `apply_calibration()` single shared policy for service + backtest.
- **Operationalization scripts** — `scripts/fit_calibration.py`, `scripts/promote_profile.py`, `scripts/report_calibration.py`.

### MLB integration

## Constraints reminder

- Do not duplicate edge, calibration, staking, or grading logic in a second path.
- Every persistence format must be versioned.
- Outcome attachment must happen after initial trace persistence, not by mutating source records.
- The sandbox engine must remain network-free.
- Anti-overengineering: does this directly prevent bad sim inputs, bad recommendations, bad replay, or bad backtests? If no, defer it.
